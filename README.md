# stock_future — 株価トレンドライン自動検出

東証プライム銘柄・主要指数・任意のティッカーの日足チャートに対して、短期/中期/
長期の 3 スケールでトレンドラインとサポート/レジスタンスを自動検出し、
Streamlit 上で可視化するツール。

各線は「検出後の将来 90 営業日でタッチ＋反発したか」（定義 B）で検証され、
チューニング済みパラメータで **全銘柄平均 80.6% の有効率** を達成している。

> 旧バージョンの LSTM 予測モデル（約 1 か月先の株価予測）は `legacy/` に退避済み。
> 本プロジェクトはそこから方針転換し、純粋にアルゴリズム的なチャート分析ツールに
> 変わっている。

```

├── requirements.txt

├── artifacts/
│   ├── trend_params.json    # チューニング結果
```

#### 銘柄選択の 4 モード (サイドバー上部)

| モード | 用途 |
|---|---|
| **東証プライム** | universe.py に登録済みの約 100 銘柄を選択 |
| **指数** | 日経 225、TOPIX ETF、S&P 500、NASDAQ、ダウなど |
| **お気に入り** | `⭐` ボタンで追加した銘柄を素早く呼び出し |
| **直接入力** | 4 桁 (7203) や yfinance 形式 (^N225, AAPL) を手入力。キャッシュに無ければ自動ダウンロード |

選択中の銘柄の右下には「⭐ お気に入りに追加 / 外す」ボタンが出る。お気に入りは
`artifacts/favorites.json` に永続化される。

#### 検出スケール

3 つのスケールを独立に ON/OFF できる:

| スケール | lookback | pivot_window | 用途 |
|---|---|---|---|
| **短期** | 90 バー (~4ヶ月) | 4 | 直近のブレイクアウト、日足スイング |
| **中期** | 240 バー (~1年) | 7 | 中期スイングトレード |
| **長期** | 500 バー (~2年) | 10 | 大きな流れ、主要な S/R |

複数スケールで同じ線が検出された場合は自動でマージされる（価格と傾きで判定）。

#### チャート構成 (4 パネル)

1. **ローソク足**: SMA 5/25/75、一目均衡表 (トグル)、検出された全線、上場来高値・安値
2. **出来高**: Vol MA20 付き
3. **RSI(14)**: 70/30 ガイドライン
4. **MACD**: ヒストグラム + MACD/Signal

土日と日本の祝日はチャートから自動的にスキップされる（gap なしで連続表示）。

#### チャートの操作

10 年分のキャッシュ全体がチャートに渡され、サイドバーの「初期表示の営業日数」
スライダーで初期ズーム範囲を決めるだけ。読み込み後は以下のように自由に
操作できる:

| 操作 | 結果 |
|---|---|
| **チャート中央をドラッグ** | 表示領域を左右にパン |
| **マウスホイール** | カーソル位置を中心に拡大/縮小 |
| **X 軸 (日付帯) をドラッグ** | 横方向のみズーム (営業日数の追加/縮小) |
| **Y 軸 (価格帯) をドラッグ** | 縦方向のみズーム (価格スケール変更) |
| ダブルクリック | 初期表示にリセット |

`uirevision` によりサイドバー操作で再描画されてもパン・ズーム位置は維持される
（銘柄を切り替える、または初期表示スライダーを動かすとリセットされる）。

#### 表示されるファンダメンタル

ヘッダーの 2 段目に PER (株価収益率)、PBR (株価純資産倍率)、配当利回りを
yfinance から取得して表示する。値は 1 時間キャッシュされ、取得失敗時は
`—` 表記となる。指数 (^N225 等) や ETF はファンダ値を持たないため `—` 表示。

#### 線の色とスタイル

| 種類 | 色 |
|---|---|
| サポート | 🟢 緑 |
| レジスタンス | 🔴 赤 |
| 上昇トレンド | 🔵 シアン |
| 下降トレンド | 🟠 オレンジ |

| スタイル | 意味 |
|---|---|
| 太い実線 | 検証パス（過去に S/R として機能） |
| 細いダッシュ | 未判定（検出時点から先のデータが不足 = 直近の線） |
| 細い点線 | 検証 NG (デフォルト非表示) |

## アルゴリズムの概要

### 検出 ([trendlines.py](trendlines.py))

1. `scipy.signal.argrelextrema` で `pivot_window` バーの局所極値を抽出
2. **水平線**: ピボットを価格でソートし、`tolerance_pct` 以内でクラスタ化
3. **斜め線**: ピボットの全ペアから直線を引き、`tolerance_pct` 以内の他の
   ピボットを数える。年率スロープ `max_slope_annual` 以下のみ受理
4. **品質フィルタ** (精度向上のキー):
   - `min_span_days`: タッチが狭い期間に集中していたら棄却
   - `max_last_touch_age_days`: 最後のタッチが古すぎる「死んだ線」を棄却
5. **スコアリング**: タッチ数 + recency ボーナス + スパン ボーナス
6. **マルチスケール**: 3 つの時間軸で独立検出、価格と傾きで重複マージ

### 検証 ([evaluator.py](evaluator.py)) — 定義 B

線の最終ピボットより *後の* 90 営業日を見て、

1. 価格がその線に 2 回以上タッチ (連続タッチは 3 日空ける)
2. うち 1 回以上は 7 日以内に逆方向へ 2% 以上反発

を満たした線だけを「有効」とカウント。フォワードデータが不足する場合 (10 バー未満)
は `None` を返し「未判定」扱いとなる（tuner はこれを集計から除外、アプリは
ダッシュ線で表示）。

`artifacts/trend_params.json`:

|---|---|
| pivot_window | 12 |
| tolerance_pct | 0.015 |
| min_touches | 3 |
| max_slope_annual | 1.0 |
| lookback_bars | 500 |
| min_span_days | 60 |
| max_last_touch_age_days | 180 |
| **precision** | **80.6%** (590 / 732 lines) |

|---|---|---|
| resistance | 40/46 = **87.0%** | 74% |

水平 S/R の精度が大きく向上した。

## 注意

- **投資助言ではありません**。過去データに対するアルゴリズム的なパターン検出ツール
- 指数 (^N225 など) は個別株より値動きが小さいため「定義 B」の 2% 反発ハードルが
  高く、個別株より未判定/NG 率が高めに出る傾向がある
- 任意ティッカー入力で初回は yfinance ダウンロードが走るため少し時間がかかる
